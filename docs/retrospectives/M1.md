## What went well

- **Three-chunk-per-large-crate cadence held.** schema (A/B/C),
  state (D/E/F), diff (A/B/C) each split into a type chunk, a logic
  chunk, and a polish chunk. Each chunk landed in a single PR with
  passing CI on first push or one tactical fix.
- **Trait-first discipline paid for itself by Week 3.** Engine tests
  that wire `MockProbe` + `MockNickel` into a real `Engine::plan` were
  a one-liner because the adapter trait surfaces were already locked.
- **The Plan §7.3 four-way discriminator landed exactly as
  specified.** Six tests in `classify::pacman::tests` cover every
  bucket (managed / to_install / forgotten / manual / adopted /
  protected) plus the priority rule (ignore > adopted > forgotten >
  manual). Get this wrong and Pearlite removes packages the user
  wants to keep — getting it right was the most consequential single
  property of M1.
- **PRD §9.3 envelope shape served unchanged.** No fields needed
  bending to match what the engine produces; `data` /
  `error` exclusivity round-trips through serde without ceremony.
- **CI tier 1+2+3 stayed green except for documented infrastructure
  fixes.** `cachyos/cachyos:latest` container, `extractions/setup-just@v3`
  regression, `cargo-machete` MSRV, and the `time 0.3.45` advisory
  were each one-PR fixes and didn't reopen.

## What didn't

- **`extractions/setup-just@v3` regression.** "no release for just
  matching version specifier" hit two PRs in a row before we swapped
  to `taiki-e/install-action` (which we already used for cargo
  tooling). Lesson: prefer one tool that handles many installs over
  many tools that handle one install each.
- **`cargo-machete` flagging speculative deps.** Three PRs (#13, #18,
  and the M1 W3 engine setup) needed follow-up commits to remove
  deps added "in case the next chunk needs them." The cure: only
  declare deps the current PR actually uses; add the others when
  they're consumed.
- **The `time = 0.3.47` MSRV jump.** RUSTSEC-2026-0009 (parse-time
  stack-exhaustion DoS) ships its fix in a release that requires
  rustc 1.86; we pin 1.85. We documented the exposure analysis (the
  DoS surface is unreachable through Pearlite's API — we never feed
  untrusted strings to `time::parse`) and ignored the advisory in
  both `audit.toml` and `deny.toml`. ADR-008 covering the MSRV bump
  policy still owes a write-up.
- **`schemars` 0.8 → 1.x bump landed mid-stream.** Should have been
  caught at chunk M1-W1-A; instead the Draft-2020-12 fixture in chunk
  M1-W1-C drove the upgrade. No actual harm — the API was forwards-
  compatible — but the bump would have been cheaper one chunk earlier.
- **Solo-dev admin merges add friction.** Each PR needs an explicit
  plain-text "yes, merge PR N with admin" because the permission
  system treats every merge as a fresh authorization. Working as
  intended; not changing the policy. But the cadence is lumpy.
